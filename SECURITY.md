# Security Policy

We take the security of our software and our clients' systems seriously. If you believe you've found a vulnerability in any MVNG Consulting repository, please report it responsibly.

## Reporting a vulnerability

**Do not open a public issue.** Instead:

- Email **security@mvng.nz** , or
- Use GitHub's [private vulnerability reporting](https://docs.github.com/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) on the affected repository (Security tab → "Report a vulnerability").

Please include:

- A description of the issue and its potential impact
- Steps to reproduce (proof-of-concept if possible)
- Affected repository, version, or commit

## What to expect

| Stage                          | Target                        |
| ------------------------------ | ----------------------------- |
| Acknowledgement of your report | within 3 business days        |
| Initial assessment             | within 10 business days       |
| Fix or mitigation plan         | communicated after assessment |

We ask that you give us a reasonable window to remediate before any public disclosure. We're happy to credit reporters who follow this process.

## Scope

This policy covers code in MVNG Consulting repositories. Vulnerabilities in third-party dependencies should be reported upstream, though we appreciate a heads-up so we can patch promptly.
